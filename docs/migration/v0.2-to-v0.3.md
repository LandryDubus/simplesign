# Migration Guide: v0.2 → v0.3

## Breaking Changes

### `SimpleSigner.SignAsync()` no longer accepts `ILogger` parameter
Use `WithLogger()` on the builder instead.
```diff
- var signed = await SimpleSigner.Document(pdf).SignAsync(logger);
+ var signed = await SimpleSigner.Document(pdf).WithLogger(logger).SignAsync();
```

### `SignatureFieldOptions.SubFilter` changed from `string` to `PdfSignatureSubFilter` enum
```diff
- options.SubFilter = "ETSI.CAdES.detached";
+ options.SubFilter = PdfSignatureSubFilter.EtsiCadesDetached;
```

### `ValidationResult` properties renamed
- `ValidationResult.IsValid` → `ValidationResult.IsSignatureValid`
- `ValidationResult.Errors` → `ValidationResult.IntegrityErrors` (use `AllErrors` for all)

### Removed APIs
- `SignerBuilder.WithVisibleSignature()` — removed; use `WithSignatureAppearance()` + `WithVisible()`

## New Features in v0.3

- **PAdES-LTA** archival timestamps via `WithArchivalTsaUrl()`
- **Batch signing** via `BatchSigner`
- **Deferred signing** via `DeferredSigner`
- **PDF/A-1b/2b/3b** detection and preservation
- **ICP-Brasil AD-RB/AD-RT** policy support
- **SHA-384/512** hash algorithm support

## Upgrade Steps

1. Update `SimpleSign` package to v0.3.x
2. Replace `ILogger` parameter with `WithLogger()` builder call
3. Replace `SubFilter` string with `PdfSignatureSubFilter` enum
4. Update `ValidationResult` property references
5. Test all signing flows end-to-end
