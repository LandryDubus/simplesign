# Migration Guide: v0.3 → v0.4

## Breaking Changes

### `CmsAttribute.CommitmentTypeIndication()` renamed to `CmsAttribute.CommitmentType()`
The old name is deprecated but still works. Update to the new name:
```diff
- CmsAttribute.CommitmentTypeIndication(CommitmentType.ProofOfApproval);
+ CmsAttribute.CommitmentType(CommitmentType.ProofOfApproval);
```

### `CryptoVerifier.VerifySignature()` no longer throws for EdDSA
EdDSA signatures are now handled via the ECDSA path when the runtime supports it. If the runtime does not support EdDSA, a `NotSupportedException` is thrown at the fallback with improved messaging.

### `PdfALevel` enum now includes PDF/A-4 values
New values: `A4a`, `A4b`, `A4u`, `A4e`. `switch` statements on `PdfALevel` must handle these new values.

## New Features in v0.4

- **SHA-3** hash algorithm support (SHA3-256, SHA3-384, SHA3-512) on .NET 9+
- **EdDSA** verification via ECDSA path (external signer pipeline only for signing)
- **PDF/A-4** (ISO 19005-4:2020) detection and preservation (experimental)
- **CAdES-XL** validation references: `CmsAttribute.CertificateRefs()`, `CmsAttribute.RevocationRefs()`, `CmsAttribute.CertValues()`, `CmsAttribute.RevocationValues()`
- **CI infrastructure**: fuzz testing, mutation testing, stress test workflows
- **Fully classified `PdfALevel` enum values** with `A2u`, `A3u`, `A4a`, `A4b`, `A4u`, `A4e`

## Upgrade Steps

1. Update `SimpleSign` package to v0.4.x
2. Add handling for new `PdfALevel` enum values in any `switch` statements
3. Update `CmsAttribute.CommitmentTypeIndication()` calls if using the old name
4. Test with `.NET 9+` runtime for SHA-3 support
