# Migration Guide: v0.4 → v0.5

## Breaking Changes

### `WithTimestamp(url, httpClient)` scope narrowed to TSA only

In v0.4, the `HttpClient` passed to `.WithTimestamp(url, client)` was used for both TSA
requests and revocation (OCSP/CRL/AIA). In v0.5, it applies **only to TSA calls** (signature
timestamp + archival DocTimeStamp).

**If you were relying on the implicit revocation scope**, add `.WithHttpClient(client)` explicitly:

```diff
  var signed = await SimpleSigner
      .Document(pdf)
      .WithCertificate(cert)
-     .WithTimestamp(tsaUrl, httpClient)
+     .WithTimestamp(tsaUrl, httpClient)
+     .WithHttpClient(httpClient)
      .WithLtv()
      .SignAsync();
```

If you pass separate clients for TSA and revocation:

```csharp
var signed = await SimpleSigner
    .Document(pdf)
    .WithCertificate(cert)
    .WithTimestamp(tsaUrl, tsaClient)         // bearer token, TSA only
    .WithHttpClient(revocationClient)          // anonymous, OCSP/CRL only
    .WithLtv()
    .SignAsync();
```

## New Features in v0.5

- **`SignerBuilder.WithHttpClient(HttpClient)`** — set the default HTTP client for
  revocation (OCSP/CRL/AIA) and TSA fallback when no TSA-specific client is configured.
- **`HttpClientFactoryProvider`** — built-in `IHttpClientFactory` adapter that implements
  `IHttpClientProvider`. Ships in `SimpleSign.Core.Http`.
- **Auto-detection of `IHttpClientFactory` in DI** — `AddSimpleSign()` now checks for
  `IHttpClientFactory` in the container and wires it automatically using
  `SimpleSignOptions.HttpClientName` as the named-client key.
- **`SimpleSignOptions.HttpClientName` is now consumed** — previously documented but
  silently ignored. Now used when `AddSimpleSign()` falls back to `IHttpClientFactory`.
- **Per-operation HTTP client slots** — TSA and revocation use independent `HttpClient`
  instances, enabling per-operation authentication.

## Fixed

- `SignerBuilder.WithHttpClientProvider` now resolves lazily (provider is called at
  signing time, not at builder-configuration time).
- Custom `IHttpClientProvider` is preserved through all builder methods (`WithLtv`,
  `WithMetadata`, `WithOperationId`, `WithPdfAPreservation`, `WithLegacyCms`,
  `WithSubFilter`, `WithArchivalTimestamp`) — no longer silently replaced by the
  static default.
- Archival DocTimeStamp (B-LTA) now uses the TSA client instead of the revocation
  client.

## Upgrade Steps

1. Update `SimpleSign` package to v0.5.x
2. If using `.WithTimestamp(url, client)` with the expectation that the client applies
   to revocation, add `.WithHttpClient(client)` after `.WithTimestamp()`.
3. (Optional) Remove manual `IHttpClientProvider` implementations — the default
   `AddSimpleSign()` now auto-wires `IHttpClientFactory` when available.
